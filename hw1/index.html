Software Development Life Cycle

SLDC Requirement Analysis & Design Development & Testing Maintenance


Overview

  The software development life cycle (SLDC) model represents squential phases that must be met to produce a time efficient, cost-efficient, and high-quality software for the customer or user. Each phases consist of deliverables required by the next phase. The phases are the following:

  1. Requirement Analysis: This phase consist of a plan, which describes the constraints.
  2. Design: This phase is a design plan based on sofware specification.
  3. Development: This phase builds the code.
  4. Testing: This phase tests for deficiencies of the software.
  5. Maintenance: This phase continously updates and advances the software.

  The SLDC consist of many models which have different utilizations. This website will describe the Waterfall model and how the sequential flow is structured.
